Confluence — The Flagship Demo

A real-time territory game that proves the thesis: when all state is a CRDT, simultaneous play creates emergent collaboration, not conflict.

Players place tokens on a shared grid. Contested cells don't use last-write-wins — they preserve both players' tokens, merged by the CRDT. The result is a living board state that reflects every player's contribution.

  • Real-time simultaneous placement — no turns
  • Contested cells merge tokens, not overwrite
  • Every move is recorded with provenance
  • Play in your browser, no installation
  • Save and resume games with IndexedDB persistence

Try It

git clone https://git.carpocratian.org/sibyl/hypertoken
cd hypertoken
npm install && npm run build

# Play Confluence in the browser
npm run relay              # Terminal 1: start relay
npm run confluence:web     # Terminal 2: build + serve
# Open http://localhost:8080 in two tabs
# Play Blackjack
npm run blackjack

# Play Cuttle
npm run cuttle

# Play Prisoner's Dilemma
npm run prisoners-dilemma

How It Works

Tokens Compose With Provenance

const enchantedSword = engine.dispatch("token:merge", {
  tokens: [sword, fireEnchantment],
  resultProperties: { label: "Flaming Sword" }
});
// enchantedSword._mergedFrom = [sword.id, fireEnchantment.id]

const pieces = engine.dispatch("token:split", {
  token: goldPile,
  count: 3
});
// pieces[0]._splitFrom = goldPile.id

State Syncs Automatically

const host = new Engine();
host.connect("ws://relay.local:8080");

const client = new Engine();
client.connect("ws://relay.local:8080");

// Both make changes → CRDTs merge → identical final state
// No conflict resolution code. It just works.

Persistence

const engine = new Engine();

// Use storage adapter (IndexedDB in browser,
// FilesystemAdapter in Node.js)
engine.useStorage(new IndexedDBAdapter());

// Save state
await engine.persist("my-game-session");

// Resume later — even after closing the browser
await engine.resume("my-game-session");

Forkable Worlds

// Snapshot current state
const snapshot = engine.snapshot();

// Explore timeline A
engine.dispatch('agent:cooperate');
const outcomeA = engine.getState();

// Rewind, explore timeline B
engine.restore(snapshot);
engine.dispatch('agent:defect');
const outcomeB = engine.getState();

What's Inside

  • 77 built-in actions — draw, shuffle, place, move, flip, merge, split, and more across 7 categories
  • Rust/WASM core — ~20x performance over pure JS, with TypeScript fallback
  • P2P networking — serverless CRDT sync (ConsensusCore) and host-authoritative mode
  • Persistence — IndexedDB (browser) and Filesystem (Node.js) adapters with auto-save
  • Browser builds — one command bundles any game for the browser
  • MCP server — LLM integration via Model Context Protocol
  • 10 example games — Blackjack, Poker, Cuttle, Confluence, Prisoner's Dilemma, Hanabi, Coup, Liars Dice, Accordion, Dungeon Raiders
  • RL interfaces — Gym and PettingZoo adapters in hypertoken-rl/ for engine-backed evaluation environments
